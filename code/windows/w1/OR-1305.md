# OR-1305 Orange Code

Orange Code exists to replace repeated human coding loops with visible, inspectable, LLM-managed work.

## Thesis

Claude Code is useful, but its session model is not enough for long-running project work. Orange Code should make the workflow itself persistent:

- one manager that understands the project,
- bounded member workers,
- supervisor review and recovery,
- persistent goals, projects, chats, and reports,
- build and capture verification,
- low-cost default testing.

## Product Direction

The first product target is not a general chat app. It is a coding workbench:

- user request in,
- manager plan and execution,
- file edits,
- build/test/capture,
- concise result,
- durable handoff for the next loop.

Chat can be the input surface, but code blocks, diffs, logs, progress, and task state must become first-class UI.

## Role Model

- `manager`: Jeong Team Lead, the primary in-app actor.
- `member`: bounded worker for a delegated task.
- `supervisor`: reviewer and recovery actor.

Use `manager` everywhere for Jeong Team Lead.

Each loop should prefer:

1. build,
2. local diagnostic,
3. mock backend capture,
4. real backend call only when the backend itself changed.
