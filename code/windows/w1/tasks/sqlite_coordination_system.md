# 작업 명세: SQLite 기반 다중 CLI 작업 조정 시스템 (OR-1328)

## 배경
기존의 파일(Markdown, JSON) 기반 작업 방식은 다중 CLI 프로세스가 동시에 접근했을 때 데이터 오염, 파일 잠금 충돌, 시간성 일관성의 문제가 있었습니다. 이를 해결하기 위해 ACID 트랜잭션을 지원하는 SQLite를 도입하여 하나의 두뇌처럼 동작하는 견고한 조정 인프라를 구축합니다.

## 목적
- 목표-프로젝트-채팅으로 이어지는 계층 구조를 명확히 정의하고 데이터 무결성을 보장합니다.
- 각 계층 별 담당자, 감독자, 멤버를 명시적으로 추적합니다.
- 실시간 활동(ActorActivity)과 결과 검토(Reviews)를 데이터베이스에 통합하여 작업 맥락을 적절히 관리합니다.

## 범위
1. 데이터베이스 엔진 통합:
   - lib/sqlite/ 에 SQLite Amalgamation 소스 포함 및 프로젝트 빌드 구성.
   - Database.h 를 통한 싱글톤 기반의 DB 연결 관리 (WAL 모드 활성화).

2. 스키마 설계 및 구현:
   - Workflow: 계층 구조, 상태(status), 진척도(progress), 담당자 ID 관리.
   - ActorActivity: 실시간 프로세스(PID) 정보, 현재 의도(intent), 집중 대상 계층 별 관리.
   - Reviews: 4자 리뷰 결과 기록.

3. Coordination 로직 고도화:
   - CCoordination 클래스의 현재 구현을 파일 API에서 SQL 쿼리로 전면 교체.
   - 기존 활동 패널 UI에서 상위 전환을 위한 데이터 매핑 계층 추가.

4. 테스트 통합:
   - 앱 시작 시 자동 DB 초기화 및 마이그레이션 로직 추가.

## 검증
- [ ] 다중 프로세스 실행 시 orange_code.db 에 동시 기록 및 조회가 충돌 없이 진행되는가?
- [ ] 데이터바인딩 계층 정리가 DB 데이터를 바탕으로 정상적으로 렌더링되는가?
- [ ] 일정 주기의 작업 결과가 DB에 기록되고 다른 프로세스에서 다시 읽히는가?
- [ ] 4자 리뷰 데이터가 DB에 정상 쓰이고 조회 가능한가?
